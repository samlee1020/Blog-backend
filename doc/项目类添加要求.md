# 后端新增项目管理功能需求

当前前端已实现 `/projects` 项目页和后台“项目管理”页面，但由于后端没有项目相关接口，前端暂时使用 `localStorage` 保存项目数据。现在需要后端新增项目数据模型和 REST API，让项目数据持久化到数据库，并支持前台展示和后台管理。

## 1. 数据模型

新增 `project` 表。

字段建议：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | BIGINT PK AUTO_INCREMENT | 项目 ID |
| title | VARCHAR(200) NOT NULL | 项目标题 |
| slug | VARCHAR(200) UNIQUE | 项目唯一标识，可选 |
| description | VARCHAR(1000) | 项目描述 |
| image_url | VARCHAR(500) | 项目展示图片 URL |
| project_url | VARCHAR(500) | 项目外部链接，如 GitHub、Demo |
| tags_json | JSON | 标签数组，如 `["React", "Compiler"]` |
| sort_order | INT DEFAULT 0 | 排序，越小越靠前 |
| status | VARCHAR(16) NOT NULL | `DRAFT` / `PUBLISHED` / `HIDDEN` |
| created_by | BIGINT | 创建管理员 |
| updated_by | BIGINT | 更新管理员 |
| created_at | DATETIME NOT NULL | 创建时间 |
| updated_at | DATETIME NOT NULL | 更新时间 |
| deleted_at | DATETIME NULL | 软删除时间 |

## 2. 枚举

新增：

```text
ProjectStatus: DRAFT, PUBLISHED, HIDDEN
```

前台只展示 `PUBLISHED` 且未删除项目。

## 3. API

统一响应结构沿用现有：

```json
{
  "code": "SUCCESS",
  "message": "success",
  "data": {}
}
```

### 3.1 前台项目列表

```http
GET /api/projects
```

权限：`PUBLIC`

查询参数：

| 参数    | 说明                    |
| ------- | ----------------------- |
| page    | 页码，从 1 开始，默认 1 |
| size    | 每页数量，默认 10       |
| keyword | 标题/描述搜索，可选     |

返回分页结构，`items[]` 为项目摘要。

只返回 `PUBLISHED` 项目，按 `sortOrder ASC, id DESC` 排序。

### 3.2 前台项目详情，可选

```http
GET /api/projects/{slug}
```

权限：`PUBLIC`

如果第一版不需要详情页，可以暂不实现。

### 3.3 后台项目列表

```http
GET /api/admin/projects
```

权限：`ADMIN`

查询参数：

| 参数    | 说明                                   |
| ------- | -------------------------------------- |
| page    | 页码                                   |
| size    | 每页数量                               |
| status  | `DRAFT` / `PUBLISHED` / `HIDDEN` |
| keyword | 搜索标题/描述                          |

返回所有未软删除项目。

### 3.4 创建项目

```http
POST /api/admin/projects
```

权限：`ADMIN`

请求体：

```json
{
  "title": "Compiler WebUI",
  "slug": "compiler-webui",
  "description": "一个在线编译器与调试工具界面",
  "imageUrl": "/uploads/other/2026/05/xx.png",
  "projectUrl": "https://github.com/example/compiler-webui",
  "tags": ["TypeScript", "WebAssembly", "Compiler"],
  "sortOrder": 1,
  "status": "PUBLISHED"
}
```

校验：

- `title` 必填，1-200 字符
- `description` 可选，最多 1000 字符
- `imageUrl` 可选，最多 500 字符
- `projectUrl` 可选，最多 500 字符
- `tags` 可选数组
- `sortOrder` 可选，默认 0
- `status` 可选，默认 `DRAFT`
- `slug` 可选，若为空可由标题生成
- `slug` 唯一，冲突返回 `CONFLICT`

### 3.5 更新项目

```http
PUT /api/admin/projects/{id}
```

权限：`ADMIN`

请求体同创建项目。

### 3.6 删除项目

```http
DELETE /api/admin/projects/{id}
```

权限：`ADMIN`

软删除，返回：

```json
true
```

### ProjectView

```json
  "status": "PUBLISHED",
  "createdAt": "2026-05-05T10:00:00",
  "updatedAt": "2026-05-05T10:00:00"
}
```

## 5. 图片上传复用

项目图片可复用现有图片上传接口：

```http
POST /api/admin/media/images
```

`usageType` 可以暂用 `OTHER`。如果后端愿意扩展，也可以新增：

```text
MediaUsageType: PROJECT
```

前端可接受两种方案：

- 复用 `OTHER`
- 新增 `PROJECT`

## 6. 前端接入目标

后端完成后，前端会把当前本地项目存储替换成真实接口：

- `/projects` 调 `GET /api/projects`
- `/admin/projects` 调 `GET /api/admin/projects`
- 新建项目调 `POST /api/admin/projects`
- 编辑项目调 `PUT /api/admin/projects/{id}`
- 删除项目调 `DELETE /api/admin/projects/{id}`

完成后项目数据应在数据库中持久化，不再依赖浏览器 `localStorage`。
